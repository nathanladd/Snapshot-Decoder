Working with Snapshots

Snapshot Decoder supports multiple snapshot formats from different ECU generations. It is certainly possible to add support for more engines in the future. Please contact me if you have any questions or suggestions. Check for updates often to see if additional snapshot types become supported.

Currently Supported Snapshot Types

Auto-Detection: Snapshot Decoder automatically detects the snapshot type when you open a file and configures the available quick charts accordingly.

V1 ECU Snapshots

1.8L, 2.4L, and 3.4L Bobcat Engine with Delphi ECU.

  • All available Snapshot header information is pulled and displayed
  • Engine hours are extracted and converted from seconds if necessary
  • MDP success rate is calculated and displayed
  • Currently 12 Quick Charts are available
  • Easily search PID descriptions and build custom line charts

V1 Engine Use Data (EUD) Snapshots

1.8L, 2.4L, and 3.4L Bobcat Engine with Delphi ECU.

  • All available Snapshot header information is pulled and displayed
  • Total engine hours and idle time are extracted, converted from seconds if necessary, and displayed seporately
  • MDP Success rate is calculated
  • Currently 4 Quick Charts are available
  • Easily search PID descriptions and build custom line charts

V2 ECU Snapshots

1.8L, 2.4L, and 3.4L Second-generation Bobcat engines with Bosch ECU.

  • All available Snapshot header information is pulled and displayed
  • Engine hours are extracted and converted from seconds if necessary
  • Currently 9 Quick Charts are available
  • Easily search PID descriptions and build custom line charts

File Formats

The following file formats are supported:

  • .xls - Excel workbook (the default format from Bobcat Engine Analyzer)
  • .xlsx - If you happen to have saved the snapshot as an .xlsx file

Opening Snapshots

  1. Click File → Open Snapshot or press Ctrl+O
  2. Navigate to your snapshot file
  3. Select the file and click Open
  4. The application will load and analyze the data

Viewing Snapshot Table Data

To see the underlying data may be useful if you get unexpected results:

  • Raw Data - Data → Raw Data shows the original imported data
  • Clean Table - Data → Clean Table shows processed data
  • PID Descriptions - Data → PID Descriptions explains each parameter
