Troubleshooting

Solutions to common issues you may encounter while using Snapshot Decoder.

File Loading Issues

Problem: "Unknown snapshot type" error when opening a file
Solution: The file may not be a valid snapshot export or may be corrupted. Ensure you're opening a snapshot file exported from a Bobcat diagnostic tool. Try re-exporting the snapshot if possible.
Be sure to capture all the engien PIDs when creating a Snapshot. Do not choose any of the diagnostic sub-categories of PIDs in Bobcat Engine Analyzer.
Problem: File opens but no data appears
Solution: Check that the snapshot file contains actual data rows. Use Data → Raw Data to view what was imported. Empty snapshots may occur if the capture was interrupted.

Snapshot Quality Issues

Sometimes snapshots may contain unexpected data or produce unusual chart results. Use the Data menu to inspect the snapshot quality and diagnose issues.

Using Raw Data Table to Diagnose Issues

The Raw Data Table (Data → Raw Data) shows the original data exactly as it was imported from the snapshot file, before any processing or cleaning.

Problem: Charts show unexpected values or missing data
Solution: Open the Raw Data Table to check the original snapshot data:
  • Missing columns - If expected PIDs are missing, the snapshot may not have been captured with all engine PIDs selected in Bobcat Engine Analyzer
  • Empty cells or zeros - Indicates the sensor was not active or not reporting during capture
  • Text instead of numbers - Some cells may contain error codes or status text instead of numeric values
  • Inconsistent data - Look for sudden jumps, repeated values, or obvious errors that indicate capture problems
  • Short snapshots - Check the number of rows to ensure the snapshot captured enough data for meaningful analysis

Using Clean Data Table to Verify Processing

The Clean Data Table (Data → Clean Table) shows the processed data after Snapshot Decoder has cleaned and converted the raw data. This is the data used for all charts.

Problem: Charts don't match expectations or show gaps
Solution: Compare the Clean Data Table with the Raw Data Table:
  • Missing rows - Some rows may have been removed if they contained invalid data
  • Converted values - Check that unit conversions were applied correctly (e.g., temperature, pressure)
  • Calculated columns - Some PIDs may be calculated from other values; verify the source data is present
  • Time or Frame column - Ensure the time/frame column is present and sequential

Common Snapshot Quality Problems

Problem: Snapshot was captured with diagnostic sub-categories instead of all engine PIDs
Solution: When creating snapshots in Bobcat Engine Analyzer, always select All Engine PIDs, not diagnostic sub-categories. Sub-category snapshots may be missing critical PIDs needed for Quick Charts and analysis. Check the Raw Data Table to see which PIDs are present, then re-capture the snapshot if necessary.
Problem: Snapshot capture was interrupted or too short
Solution: Open the Clean Data Table and check the number of data rows. Short snapshots (fewer than 50-100 rows) may not capture enough information for meaningful analysis. For events like cranking or acceleration, ensure the snapshot capture spans the entire event plus some time before and after.
Problem: Data shows flat lines or no variation
Solution: Use the Clean Data Table to verify the PID values:
  • If all values are identical, the sensor may not have been active during capture
  • If values are all zero, check the Raw Data Table to see if this is the original data or a conversion issue
  • Some parameters only change under specific conditions (e.g., EGR only operates when engine is warm)
Best Practice: Before analyzing a snapshot, quickly review both the Raw Data Table and Clean Data Table to verify data quality. This helps identify capture issues early and saves time troubleshooting unexpected chart results.

Chart Issues

Problem: A Quick Chart button does nothing when clicked
Solution: Quick charts are only enabled when the required PIDs are present in the snapshot.
For example, if the engine is not equipted with an intake throttle valve, clicking the air throttle Quick Chart button will do nothing.
Problem: Chart appears blank or shows a flat line
Solution: This typically means the parameter had no variation during capture. Try zooming in to see small changes, or check if the sensor was active during the snapshot. Use Data → Clean Table to verify the actual PID values.
Problem: Chart zoom is not working properly
Solution: Click the Home button in the toolbar to reset the view. If zoom behavior seems erratic, try clicking the zoom tool again to re-select it.

Display Issues

Problem: Text or buttons appear cut off
Solution: Try maximizing the window or increasing screen resolution. The application works best at 1280x720 or higher resolution.
Right-click your Desktop and choose Display Settings. Under Scale & Layout make sure Scale is 100%.
Problem: Charts look blurry or pixelated
Solution: On high-DPI displays, ensure Windows display scaling is set to 100%. Unusual scaling values may affect rendering.

Getting More Help

If you continue to experience issues:

  • Check for application updates via Help → Update
  • Contact support with details about the issue and steps to reproduce
  • Include the snapshot file if possible when reporting issues
